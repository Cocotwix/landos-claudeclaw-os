# landos-plan

Create a focused implementation plan for one LandOS task.

## Plan requirements

- Goal
- Files likely touched
- No-go rules
- Tests
- Acceptance criteria
- Commit scope

## Rules

- Keep the plan narrow and executable.
- Do not mix unrelated work into one plan.
- Do not plan commits or pushes unless Tyler asks for that separately.
- Do not include secrets, tokens, or private data.
